# 四设备马尔可夫随机博弈实验报告

## 1. 报告目的

本文总结当前项目在两种拓扑结构下的实验结果，对比四种算法在无线随机接入任务中的表现，并给出结果解释与后续建议。

本次对比的两种拓扑为：

- `all_to_all`：全连接拓扑
- `star`：星形拓扑

本次对比的四种算法为：

- `value_iteration`
- `IQL`
- `QMIX`
- `MAPPO`


## 2. 实验设置

### 2.1 环境设置

实验环境来自 [env.py](/Users/yangwenhaomac/Desktop/Tele/Markov/wireless_marl/env.py)。

核心设定如下：

- 设备数：`4`
- 每回合时长：`200` 步
- 信道状态：`空闲 / 拥堵` 两态
- 本地缓冲：二值状态
- 观测维度：每个设备仅观测 `[信道观测, 本地缓冲]`
- 碰撞规则：同一时隙只要 `k_tx >= 2` 就判定为碰撞
- 成功规则：只有 `k_tx == 1` 且目标合法时，才可能成功发送
- 拓扑作用：仅用于限制“目标动作是否合法”

这意味着当前环境中的拓扑差异，主要体现在“某些发送目标是否无效”，而不是链路容量、接收端竞争、路径长度或空间复用。

## 4. 分算法分析

### 4.1 值迭代

`value_iteration` 在两种拓扑下都取得了最高吞吐量 `0.7390`，且碰撞率为 `0`。

这说明：

- 当前任务规模较小，集中式精确规划器可以直接逼近全局最优
- 在当前环境建模下，拓扑限制并没有改变理论最优值
- 值迭代应被视为本任务的“上界基线”或“oracle baseline”

结论：

- `值迭代` 是本实验中的最优方法
- 它适合在汇报中作为“理论最优参考线”


### 4.2 IQL

`IQL` 在两种拓扑下表现非常接近：

- 全连接：`0.3938`
- 星形：`0.3922`

优点：

- 最终碰撞率为 `0`
- 吞吐量稳定达到 `0.39` 左右
- 实现简单，训练成本低

缺点：

- 训练过程波动较明显
- 从训练日志看，后期仍会阶段性退回到高碰撞或过度保守策略

结论：

- `IQL` 是一个性价比较高的轻量基线
- 它适合作为“简单可解释”的学习型算法展示


### 4.3 QMIX

`QMIX` 在两种拓扑下也表现稳定：

- 全连接：`0.3820`
- 星形：`0.3777`

优点：

- 碰撞率始终为 `0`
- 奖励接近 `IQL`，能耗略低
- 训练后期非常稳定，曲线明显比 `IQL` 平滑

相对 `IQL` 的特点：

- 吞吐量略低一点
- 平均奖励略高或相当
- 能耗更低
- 收敛后更稳健

结论：

- `QMIX` 是当前四种方法中最强的学习型团队方法之一
- 如果要在 demo 里突出“多智能体协作学习”，`QMIX` 比 `MAPPO` 更适合作为主展示算法


### 4.4 MAPPO

`MAPPO` 是本次实验中表现最弱、同时对拓扑最敏感的算法。

结果如下：

- 全连接：`throughput = 0.2573`
- 星形：`throughput = 0.0552`

同时：

- 全连接下仍有明显碰撞：`0.0622`
- 星形下虽然碰撞率下降到 `0.0095`，但平均奖励已经为负

这表明它在星形拓扑下几乎退化为“过度保守策略”：

- 不太敢发，导致吞吐量极低
- 发送一旦发生，又容易因为动作分布里残留无效选择而受罚

结论：

- 当前参数配置下，`MAPPO` 不适合作为主要演示算法
- 它更适合作为“为什么 on-policy 方法在该任务里不占优”的反例


## 5. 分拓扑分析

### 5.1 全连接拓扑

全连接下的性能排序为：

`值迭代 > IQL ≈ QMIX > MAPPO`

特点：

- `IQL` 和 `QMIX` 都能学到无碰撞的有效发送策略
- `MAPPO` 虽然能工作，但明显弱于前两者
- 全连接拓扑没有对动作合法性形成强约束，因此学习难度较低


### 5.2 星形拓扑

星形下的性能排序为：

`值迭代 > IQL ≈ QMIX >>> MAPPO`

特点：

- `IQL` 和 `QMIX` 几乎没有明显退化
- `MAPPO` 出现断崖式下降
- 说明星形拓扑下“动作合法性约束”对 `MAPPO` 影响远大于对 `IQL/QMIX` 的影响


### 5.3 两种拓扑的差异总结

当前环境下，拓扑对前三种算法的影响非常小，对 `MAPPO` 的影响非常大。

这是一个很重要的结论。

原因不是“星形拓扑本身没有差异”，而是当前环境中拓扑的作用方式比较弱：

- 拓扑只限制目标是否合法
- 发送成功与否不依赖目标节点类型
- 不存在接收端容量竞争
- 不存在链路质量差异
- 不存在多跳路由成本

因此，只要算法能学会：

- 每步最多一个设备发送
- 且发送到一个合法目标

那么无论是全连接还是星形，都能达到很相近的效果。


## 6. 为什么会出现这样的结果

### 6.1 为什么值迭代在两种拓扑下几乎完全一致

因为当前环境中的最优策略本质上是“协调轮流发送，避免碰撞”。在这个前提下：

- 每个设备在两种拓扑下都至少存在一个合法发送目标
- 成功发送的收益不依赖目标是谁

所以拓扑并没有改变全局最优策略的价值结构。


### 6.2 为什么 IQL 和 QMIX 差别不大

因为这个任务的核心难点并不是高维协同，而是“避免多人同时发”。一旦学会这一点：

- `IQL` 即使只基于局部观测，也能得到一个可用策略
- `QMIX` 虽然更强，但未必能在当前简化环境中体现出大幅优势

也就是说，当前环境对 `QMIX` 的团队建模能力利用得还不够充分。


### 6.3 为什么 MAPPO 在星形下表现很差

主要有三点原因：

- 星形拓扑下，叶节点只有一个合法目标，其余发送动作会触发 `invalid` 罚分
- 当前 `MAPPO` 使用共享 actor，而中心节点与叶节点的决策角色并不对称
- `MAPPO` 是按策略分布采样评估的，只要动作分布里保留了无效动作概率，星形拓扑就会持续放大这个问题

因此它在星形下会变成一种“低碰撞但几乎不发”的策略，表现为：

- 碰撞率低
- 吞吐量更低
- 奖励甚至为负


## 7. 对 demo 展示的建议

### 7.1 推荐展示顺序

建议在前端或汇报中使用以下顺序：

1. `值迭代`：作为理论最优上界
2. `QMIX`：作为最稳健的学习型协作算法
3. `IQL`：作为轻量级学习基线
4. `MAPPO`：作为对比用弱基线


### 7.2 推荐主讲结论

可以直接使用以下表达：

- 在当前四设备随机接入任务中，集中式精确规划仍然是最优参考。
- 在学习型方法中，`QMIX` 与 `IQL` 都能稳定学出接近 `0.39` 的吞吐量，并将碰撞率压到 `0`。
- `QMIX` 更稳，`IQL` 更简单。
- `MAPPO` 对当前任务和星形拓扑不够适配，表现明显弱于前两者。


### 7.3 当前环境的解释边界

如果要讨论“拓扑结构对系统性能的影响”，需要提醒听众：

- 当前环境中的拓扑影响偏弱
- 它更像是在比较“动作合法性约束”，而不是比较真实网络中的链路结构差异

因此本实验更适合说明：

- 不同多智能体算法在同一接入控制问题上的性能差异

而不太适合直接说明：

- 全连接网络一定比星形网络更优


## 8. 后续改进建议

如果后续希望让“拓扑差异”更明显，建议修改环境：

- 为不同链路设置不同成功概率
- 引入目标节点接收竞争或中心节点拥塞
- 为非法或低质量链路设置更强惩罚
- 引入多跳转发或中继代价
- 将奖励与目标节点、链路质量、队列积压关联起来

如果后续希望提升 `MAPPO`，建议：

- 改为不共享 actor
- 增加中心节点与叶节点的角色区分输入
- 调整熵系数与学习率
- 延长训练轮数并加入更稳定的 advantage 归一化与 value 约束


## 9. 报告结论

本次实验的结论可以总结为三条：

1. `值迭代` 是当前任务中的最优参考方法，在两种拓扑下都显著优于学习型算法。
2. `IQL` 与 `QMIX` 在当前环境下都表现稳定，且两者在全连接和星形拓扑下差异很小。
3. `MAPPO` 对当前任务尤其是星形拓扑不适配，表现明显弱于其他三种方法。

就当前项目的演示和答辩用途而言，推荐把 `值迭代 + QMIX + IQL` 作为主要展示对象，把 `MAPPO` 作为补充对照组。
